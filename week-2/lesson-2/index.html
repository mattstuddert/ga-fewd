Default browser styles

Browsers typically have default styles that they apply to certain elements. Unfortunately, these styles are sometimes different for each browser.

This means that if you are not 'normalising' your styles before starting to write code, you may run into difficulties making everything look the same across browsers.

Time to level the playing field!

Positioning

Normalize.css vs reset.css

Normalize.css maintains some common default styles for certain elements (heading sizes etc), but normalises necessary styles across all browsers.

Reset.css completely resets all elements' styles, so that all elements are starting from the same base level.

Positioning

Positioning

Having a good understanding of CSS positioning is crucial for effective layout of a page. However, it is an often misunderstood property, that can quickly cause issues if used incorrectly.

Positioning

Position properties

The position property can have one of 5 possible values:

  • Static (default)
  • Relative
  • Absolute
  • Fixed

Positioning

position: static;

This position declaration is the default value on all elements, if no other position value has been set. It contains the following characteristics:

  • Element maintains normal behaviour, sitting in the flow of the document.
  • Does not respond to top, right, bottom, left or z-index properties.

Positioning

position: relative;

  • If no other properties are set, the element will maintain all normal behviour.
  • Does respond to top, right, bottom, left and z-index properties.
  • If one of these other properties are set, the element will change position on screen, but the original position of the element will remain, as if the element was still there.

Positioning

position: absolute;

  • Pulls the element out of the normal flow of the document. Meaning other elements will fill the space left by the absolutely positioned element.
  • Block-level elements no longer stretch to fill the width of the container.
  • Does respond to top, right, bottom, left and z-index properties.
  • Is positioned relatively to its closest ancestor with a position property other than position: static;.

Positioning

position: fixed;

  • Pulls the element out of the normal flow of the document. Meaning other elements will fill the space left by the fixed position element.
  • Block-level elements no longer stretch to fill the width of the container.
  • Does respond to top, right, bottom, left and z-index properties.
  • Is positioned relatively to the screen's viewport.

Positioning

z-index

The z-index property specifies the z-order of an element and its descendants.

This relates to how elements are 'stacked' in the viewport and therefore which elements should appear closer to the user and sit on top of other elements.

Positioning

z-index (cont.)

Remember, a position value other than static has to be set in order for z-index to work.

The z-index value can literally be any number you like. So, as you can imagine, it can get messy very quickly. Always be sure to define a z-index scale for a project to manage this.

Presenting your content in the best way possible is a key element to a site's success. CSS can help you with that!

Fonts

Font properties

  • font-family: specifies the type of font you would like to use. Can have multiple values, separated by commas, to provide fallbacks.
  • font-size: specifies the size of the font. Can be absolute or relative values.
  • font-weight: specifies the boldness of a font. Can use keywords or numerical values.
  • line-height: specifies the minimum height of the line boxes within an element. Can take length and percentage values, as well as unitless values e.g. line-height: 1.7;

Content

Content properties

  • text-align: often placed on the parent container to determine how its inline content is aligned.
  • letter-spacing: allows control of spacing between characters.
  • text-transform: specifies how to capitalise an element's text. Values include uppercase, lowercase and capitalize.
  • text-decoration: alters the formatting of text, allowing values such as underline, overline and line-through.

Key Takeaways

  • We use normalize.css or reset.css to normalise default browser styles

  • The position property is used to change the position of elements to either static, relative, absolute or fixed

  • Changing the font and how text is displayed can have a dramatic impact on how readable your content is

  • Font icons can be used to add icons to your site in the form of SVGs, which are great as they can scale without losing resolution
