Forms

Getting data from users

Forms are an essential part of the internet. It's how sites gather data from their users.

Examples of form usage include collecting billing information, sign up details and performing a site search, among many others.

Creating a form

All form input elements should be wrapped inside a form tag.

Forms

Form action

The action attribute on a form points to the URL that the data is supposed to be sent to.

Forms

Form method

The method attribute on a form relates to the HTTP method used to submit the form data.

The HTTP methods used in HTML forms as the method attribute are GET or POST.

Inputs

Forms are made up of input fields. The type of an input is then defined using the type attribute.

Forms

Input Types

HTML input types include:

  • Text
  • Number
  • Email
  • Password
  • Date
  • Radio
  • Checkbox
  • Range

Forms

Text

Text input type is the most commonly used. It is used for short text inputs.

Forms

Number

Number input type enforces a numerical input from the user, disallowing letters.

Forms

Email

Email input type has built in HTML form validation, which kicks in if the input does not match an valid email format when the form is submitted.

Forms

Password

Password input types obscures the character input, to keep the input hidden.

Forms

Date

Date input types allow for a formatted date inputs, providing a useful pre-styled calendar.

Forms

Radio

Radio buttons allow for the user to select a single value from a group of pre-defined options. The value attribute must be set on each radio input.

Forms

Checkbox

Checkbox inputs allow for the user to select multiple values from a group of pre-defined options.

Forms

Range

Range inputs render a slider that the user can drag to select the value they want. Both min and max attributes can be set to declare the value range.

Forms

Hidden Input

Another type of input is a hidden input. This allows data to be submitted with the form, whilst not being rendered to the screen. The data being submitted is passed in to the value attribute.

Other Form Elements

Beyond the different type attributes that are available on the input element, there are other form elements that can define how a user can input data. These include:

  • Textarea
  • Select

Textarea

A textarea can be used to allow for longer text inputs from the user.

Forms

Select

Select menu allows the user to make a selection from a dropdown list of values.

Forms

Label

Label elements are used to prompt the user as to what information the input expects.

Forms

Button

A button, placed inside a form, submits the form when clicked by default.

Forms

Fieldset

Fieldset elements wrap label and input elements to group set of inputs under a common theme.

Forms

Legend

A legend can be used in conjunction with a fieldset to namespace the group of inputs.

Another attribute that form inputs require is the name attribute. This defines the name of the field for when the data is pushed to a database.

Disabled Attribute

A disabled attribute can be added to elements to disallow interaction from the user.

A placeholder attribute can be added to form fields to help prompt the user as to how the field should be filled out.

Key Takeaways

  • Forms allow users to input information, which can then be saved to a database, or used to perform actions, such as verify the user or run a search

  • Forms are made up of form fields, which can be defined using elements such as input, textarea and select

  • HTML attributes are used to define critical information on inputs, such as type and name

  • Google Forms provides a free and easy way to collect data from users on a static site
