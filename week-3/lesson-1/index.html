Lesson 4

CSS display

Elements typically have some default styles before any code is even written.

Some elements have more default styles than others, however all elements fall into one of two categories: 'block' or 'inline' elements.

Lesson 4

Block-level elements

Block-level elements are elements that, when no width is set, will expand to fit the length of their container. This means they'll try to occupy the whole row...they're greedy!

Block-level elements have the following characteristics:

  • When no width is set, will expand to the width of its container.
  • margin and padding push elements away both horizontally AND vertically.

Lesson 4

Block-level element examples

<div>, <section>, <header>, <footer>, <ul>, <ol>, <li>, <table>, <fieldset>, <figure>, <aside>, <article>, <nav>, <blockquote>

Lesson 4

Inline elements

Inline elements fit perfectly in the flow of content and allow multiple elements on a row. This makes them perfect for use within content. They're also useful when you want elements to sit next to each other on a row.

Inline elements have they following characteristics:

  • Sit in the flow of content.
  • margin and padding only push elements away horizontally.
  • Do not respond to set height or width.

Lesson 4

Inline element examples

<span>, <em>, <i>, <b>, <strong>, <img>, <a>, <input>, <button>, <label>, <select>, <abbr>, <textarea>

Lesson 4

Inline-block elements

As we saw in the example, elements with display: inline-block; contain characteristics from both block-level and inline elements.

Here's a breakdown of those characteristics:

  • Sit in the flow of content, if no margins or padding have been set.
  • margin and padding push elements away both horizontally AND vertically.
  • Responds to set height and width.

Lesson 4

Hiding elements

Sometimes we might want to hide elements on the page. An example of one of these times might be when there is a dropdown menu that we want to hide until the user hovers over a menu item.

Lesson 4

display: none; vs visibility: hidden;

When elements are inserted into the DOM Tree, they are being 'parsed'. But before they are actually placed on the page, they need to be 'rendered'.

Lesson 4

display: none;

Elements with display: none; are parsed, but not rendered. Meaning that surrounding elements act like there is no element present.

Lesson 4

visibility: hidden;

Elements with visibility: hidden; are parsed AND rendered. This means that the hidden element still occupies the space it otherwise would have.

Flexbox

Lesson 4

What is Flexbox?

Flexbox is a new module in CSS3 that allows us to very easily align elements, order and re-order and change direction of presentation.

It removes the need for floats with much more readable and maintainable code.

Lesson 4

Flex Container

The main principle behind Flexbox is that the container has the ability to expand and shrink child elements to fit the available space.

Lesson 4

display: flex;

To create a flex container, we simply apply either display: flex; or display: flex-inline; to the parent element.

Lesson 4

Flexbox Container Properties

Lesson 4

Flex Container Properties

Once display: flex; has been assigned to the container element, a number of new, Flexbox-specific CSS properties become available to use. Below are the properties that can be applied to the container itself:

  • flex-direction
  • flex-wrap
  • justify-content
  • align-items
  • align-content

Lesson 4

flex-direction

flex-direction dictates the direction in which the container's items will flow.

Lesson 4

justify-content

justify-content defines how the browser distributes space around and between a flex container's items on the main axis.

Lesson 4

align-items

align-items defines how browsers distribute space around and between elements on the cross axis. So think of it as justify-content, but in the perpendicular direction.

Lesson 4

flex-wrap

flex-wrap defines how elements wrap when there is no more space left on the row.

Lesson 4

flex-flow

flow-direction and flex-wrap are used so frequently together that flex-flow can be used as a shorthand.

Lesson 4

align-content

align-content works in a similar way to justify-content, defining the space around rows of flex items on the cross axis.

Lesson 4

Flexbox Item Properties

Lesson 4

Flex Item Properties

Flex items are the elements that live within a flex container. There are specific CSS properties that can be applied to flex items, which can be seen below:

  • align-self
  • order
  • flex-grow
  • flex-shrink
  • flex-basis
  • flex

Lesson 4

align-self

align-self works in exactly the same way as align-items, but is applied to single items within the flex container.

Lesson 4

order

order allows you to apply order values to your flex items, which will affect the order in which the elements are shown.

Lesson 4

flex-grow

flex-grow makes flex elements grow into the available space around them.

Lesson 4

flex-basis

flex-basis is the Flexbox way of defining an item's width.

Lesson 4

flex-shrink

flex-shrink determines how an element shrinks in relation to other elements when there is not enough space on a row.

Lesson 4

flex

flex is the shorthand to combine flex-grow, flex-shrink and flex-basis on one line.

Review

Key Takeaways

  • Elements can either be block-level elements or inline elements. There is also a third display property of inline-block that acts as a hybrid.

  • Flexbox has really changed the way that frontend developers can think about layouts.

  • By providing fine-grained control over element positioning and spacing, Flexbox has solved many common previous CSS issues.
