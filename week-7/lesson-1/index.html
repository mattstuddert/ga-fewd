Responsive Basics

Fixed Layout

A fixed layout means that the layout will never change, regardless of the screen size the user is viewing the site on.

Examples include:

  • Google
  • UPS
  • Amazon

Responsive Basics

Fixed Layout Features

  • Sites with a fixed layout will include a container with a set width, often 960px. The width of this container does not change.

  • Because of it's fixed nature, pixels are often used when deciding which unit to add to declarations such as width, height, margin, padding etc.

Responsive Basics

Adaptive Layout

Adaptive layouts typically contain multiple set 'breakpoints' that, when hit, change the layout to adapt to the screen size.

Responsive Basics

Responsive Layout

Responsive layouts continually change according to the screen size of the of the user's device.

Examples include:

  • General Assembly
  • BBC
  • Techcrunch

Responsive Basics

Responsive/Adaptive Layout Features

  • Responsive layouts include different layouts depending on the screen size.

  • Column widths in the grid are sized as percentages.

  • EMs/REMs can be used to create cascading size changes based on the user's screen size.

Responsive Basics

Mobile Specific Sites

In the early days of the mobile web, companies often created a mobile specific version of their site. If the user navigated to their site on a mobile device, it would show the mobile version of the site.

Responsive Basics

Mobile Specific Drawbacks

  • Adds the complexity of a separate codebase

  • Dilutes SEO authority

  • Makes sharing links with desktop users a frustrating experience

Examples include:

Responsive Basics

Logical Operators

Logical operators and, only and not can be used to further define media queries.

Responsive Basics

Media Features

The most common conditions used in media queries are to do with media features. They include height, width, device-height, and device-width.

These media features can be prefixed with min or max to determine range.

Responsive Basics

Using min and max

  • min defines values that are equal to or greater than the given value.

  • max defines values that are equal to or less than the given value.

Mobile First vs Desktop First

Responsive Basics

Desktop First

A desktop first approach styles all desktop styles and then adds tablet/mobile styles inside max-width media queries.

Responsive Basics

Mobile First

A mobile first approach defines mobile styles outside of the media queries and progressively loads in styles for larger screens using the min-width declaration.

Responsive Basics

Mobile First Benefits

  • Progressively load in styles for larger screens, saving bandwith
  • Forces consideration of mobile users from day one
  • Forces designer to define a clear content hierarchy
  • Control the loading of larger assets for desktop
  • Easier to work mobile-out as a developer

Review

Key Takeaways

  • Responsive design is a crucial aspect of the modern web.

  • Designing and building mobile first is a preferable workflow for creating great web experiences at every screen size.

  • Relative units are great to use when building a responsive site.

  • Consider changing your media depending on the user's screen size. For example, smaller images for mobile.
