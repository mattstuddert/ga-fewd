CSS units

There are many different units that can be used to declare CSS properties such as height, width, margin, padding, plus many more.

Responsive Basics

CSS lengths (cont.)

The different types of CSS length units include:

  • Absolute units
  • Relative units
  • Viewport percentage lengths
  • Percentage

I will only be touching on the main units. If you would like to read up on them all, then here is a great article written by Chris Coyier.

Responsive Basics

Absolute units

By far the most commonly used absolute unit is the pixel (px).

Once an element has an explicit pixel width or height assigned to it, it will never change size based on unit changes elsewhere in the code or based on screen size.

Responsive Basics

Relative units

The two most common relative units are em and rem units.

em units are calculated as a multiple of the closest ancestor with an explicit font-size declaration.

rem units are expressed as a multiple of the root element (the <html> element).

Responsive Basics

Viewport percentage lengths

vh and vw can be used to set an element's size relative to a percentage of the viewport. Both are used by writing a value of between 0-100 before expressing the unit e.g. 70vh.

vh is a percentage value of the height of the viewport.

vw is a percentage value of the width of the viewport.

Responsive Basics

Percentage

Element dimensions can also be declared as a %.

An element's percentage will define its dimensions in relation to its parent element.

For example, if the parent container has a width of 500px and the child's width is set to 50%, the width of the child will be 250px.

Responsive Basics

Fixed Layout

A fixed layout means that the layout will never change, regardless of the screen size the user is viewing the site on.

Examples include:

  • Richard Hardie Car Sales
  • The Masters Law Firm

Responsive Basics

Fixed Layout Features

  • Sites with a fixed layout will include a container with a set width, often 960px. The width of this container does not change.

  • Because of it's fixed nature, pixels are often used when deciding which unit to add to declarations such as width, height, margin, padding etc.

Responsive Basics

Adaptive Layout

Adaptive layouts typically contain multiple set 'breakpoints' that, when hit, change the layout to adapt to the screen size.

Responsive Basics

Responsive Layout

Responsive layouts continually change according to the screen size of the of the user's device.

Examples include:

  • General Assembly
  • BBC
  • Techcrunch

Responsive Basics

Responsive/Adaptive Layout Features

  • Responsive layouts include different layouts depending on the screen size.

  • Column widths in the grid are sized as percentages.

  • EMs/REMs can be used to create cascading size changes based on the user's screen size.

Mobile Specific Sites

In the early days of the mobile web, companies often created a mobile specific version of their site. If the user navigated to their site on a mobile device, it would show the mobile version of the site.

Mobile Specific Drawbacks

Media Features

The most common conditions used in media queries are to do with media features. They include height, width, aspect-ratio, and resolution.

These media features can be prefixed with min or max to determine range.

Responsive Basics

Using min and max

  • min defines values that are equal to or greater than the given value.

  • max defines values that are equal to or less than the given value.

Mobile First vs Desktop First

Responsive Basics

Desktop First

A desktop first approach styles all desktop styles and then adds tablet/mobile styles inside max-width media queries.

Responsive Basics

Mobile First

A mobile first approach defines mobile styles outside of the media queries and progressively loads in styles for larger screens using the min-width declaration.

Responsive Basics

Mobile First Benefits

  • Progressively load in styles for larger screens, saving bandwith
  • Forces consideration of mobile users from day one
  • Forces designer to define a clear content hierarchy
  • Control the loading of larger assets for desktop
  • Easier to work mobile-out as a developer
