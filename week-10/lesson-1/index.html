HTML Tables

Historical layouts

In the days before smartphones and tablets, websites were typically built using tables. This allowed for structured fixed layouts.

But it is hard to re-format a table's layout, so developers moved away from this technique once accessing web pages from different sized devices became the norm.

HTML Tables

Modern day usage

The move away from tables for layout purposes has meant that tables can be used for what they were originally intended to be used for...

...presenting tabular data!

HTML Tables

Table basics

Tables are structured using columns and rows in order to present data to the user.

The core HTML elements for a table are <table>, <tr> (table row) and <td> (table data).

Table rows contain the table data elements, which make up the columns.

HTML Tables

The first element in any HTML table is the <table> element. This wraps all other table-related HTML elements.

HTML Tables

<tr> tags then define each table row.

HTML Tables

Within the <tr> tags sit the <td> tags, which each define a new column.

Corresponding <td> tags within a row sit in the same column.

HTML Tables

Table headings

If needed, headings can be added to provide context to the data. The <th> tag is used for this and they sit within a row.

HTML Tables

Table caption

To add a caption or title to a table, the <caption> element can be placed just inside the opening <table> tag.

HTML Tables

Table structure

More structure can be added inside the <table>, with the addition of <thead>, <tbody> and <tfoot> tags.

HTML Tables

Table head

The <thead> tag can be added to separate the head row of the table.

HTML Tables

Table body

The <tbody> tag is used to wrap the body of the data.

Table foot

The <tfoot> tag is used finally to present any extra data.

UI Frameworks

UI Frameworks

Simplifying Development

UI frameworks aim to simplify the development process by offering reusable classes and components.

This is great for rapid prototyping and can also help build highly maintainable projects by offering a set of guidelines and best practices to work to.

UI Frameworks

Simplifying Development

Each framework has its own approach on how to build out websites, which can help developers build responsive websites without deep technical and design skills.

We'll take a look at:

  • Bootstrap
  • Foundation
  • Bulma
  • Tailwind CSS

UI Frameworks

Bootstrap

Started as an initiative at Twitter, Bootstrap has grown to be the most used UI framework in web development.

It includes both CSS and JS based components for rapid development, along with resuable classes for finer control of the layout.

UI Frameworks

Bootstrap

Pros

  • Great for rapid prototyping
  • Large ecosystem of layouts, themes and UI components
  • LESS and SASS support


Cons

  • Websites look very similar
  • Customisation can get tricky

UI Frameworks

Foundation

Foundation and Bootstrap have battled it out for dominance in the UI framework space since they were both first created.

Like Bootstrap, Foundation offers a set of pre-built components that you can use for rapid development and prototyping.

UI Frameworks

Foundation

Pros

  • Offers flexibility for UI customisation and easy options to customise the install itself.
  • Large library of accessible, pre-built components
  • SASS support
  • Lots of training and tutorials to learn from


Cons

  • Quite complex when compared to other UI frameworks

UI Frameworks

Bulma

Bulma is a CSS-only framework based on Flexbox. It makes building responsive websites extremely easy and offers well-named, reusable classes to use throughout your markup.

UI Frameworks

Bulma

Pros

  • Extremely easy to get up and running with
  • Fully-customisable with SASS integration and variable overrides included


Cons

  • JavaScript needs to be added on for a number of the components

UI Frameworks

Tailwind CSS

Tailwind is a utility-first CSS framework, which means it provides a library of reusable classes that have extremely specific roles. The addition of these classes to HTML elements combine to customise both groups of elements and individual elements.

UI Frameworks

Tailwind CSS

Pros

  • Extremely customisable interfaces through utility classes
  • Don't need to invent random class names and your CSS stops growing
  • Can abstract common styles to components to simplify markup


Cons

  • Steeper learning curve than other frameworks
  • JS needs to be added to make interactive components work

Review

Key Takeaways

  • HTML tables are used to present tabular data to your users

  • Tables consist of rows (<tr>) and columns (<td>)

  • For structuring tables, the <thead>, <tbody> and <tfoot> tags can all be used

  • UI frameworks can be used to simplify and speed up UI development.
